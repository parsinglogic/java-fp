Package dev.javafp.tree

Class ImTreeZipper<A>

java.lang.Object
dev.javafp.tree.ImTreeZipper<A>
public class ImTreeZipper<A> extends Object

A zipper on an ImTree .

One way to think about zippers is that they are the im equivalent of java.util.ListIterator - they allow bi-directional navigation and "mutation" of the underlying collection.

Another way to think about zippers is that they are a way of referring to a part of an im collection in the context of the whole collection - this allows us to easily create a new collection with just that part of it modified.

Another other way to think about zippers is that they are a breadcrumb trail from a part of a data structure up to the root of that data structure.

Zippers were first mentioned in a paper by Gerard Huet in 1997 (Huet, Gerard (September 1997). "Functional Pearl: The Zipper". Journal of Functional Programming 7 (5): 549554.) and have become an important concept in functional programming.

Consider this tree: a /
b c /
d e / f

a, b, c etc are the elements at each node. Let's denote the tree with the root element of a as A etc so that we can distinguish between elements and trees.

Then lets look at the zipper that is focussed on F 


 (focus, side, parent)

              a     zA = (A, null, null)
             / \
            b   c   zB = (B, left, zA)
           / \
          d   e     zE = (E, right, zB)
             /
            f       zF = (F, left, zE)

In the example above, lets 'modify' our focus node F and set the element to g

We get a new zipper, It points to the same parent but its focus element is different 


            g      zG = ( G, left, zE)

Now let's go up: 


            e       zE' = (E', right, zB)
           /
          g

We have created a new tree, E', containing e at the root and with a left child of g and a right child of Nil

Now let's go up again: 


            b        zB' = (B', left, zA)
           / \
          d   e
             /
            g

Again we have created a new tree, B' containing b at the root with a left child of d and a right child of E'

Finally, we go up again: 


              a     zA' = (A', null, null)
             / \
            b   c
           / \
          d   e
             /
            g

We have now created a new tree that is the same as the old one except that it has the element g where element f used to be.

ImTreeZippers work on trees that are being rebalanced
