Class ImRoseTreeZipper<T>

java.lang.Object
dev.javafp.tree.ImRoseTreeZipper<T>
All Implemented Interfaces:
FIterator<ImRoseTree<T>>
public class ImRoseTreeZipper<T> extends Object implements FIterator<ImRoseTree<T>>

A zipper on an ImRoseTree lets you navigate back and forth in the tree and "mutate" it as you go.

Introduction

In general, zippers on immutable data structures allow you to move around the data structure, making local modifications efficiently and only making the complete new structure when you use close().

Recall that a rose tree has a root node containing an element and a list of sub-trees. The definition is recursive - each sub-tree also has an element and a list of sub-trees and so on. Leaf trees are those with no sub-trees - ie their list is empty. 


 data RoseTree = RoseTree a [RoseTree a]

A zipper on a rose tree with focus at node n is recursively defined as:

  1. a list zipper on n 's parent sub-trees, focussed on n

  2. a reference to the rose tree zipper for n 's parent.

The first list zipper as defined above is called the current list zipper.

In this tree: 


 1
 .......
 2 3   6
   ...
   4 5

a zipper on the node with element 5 would be:

  1. a list zipper on list [4, 5] with the focus on 5 and a reference its parent which is

  2. a list zipper on list [2, 3, 6] with the focus on 3 and a reference its parent which is

  3. a list zipper on list [1] with the focus on 1 and no parent

In this example the current list zipper is 1. above.

(The list zipper on the root is really an implementation detail - we consider the root tree to be in a list of size 1)

The toString() method would show it like this: 


 [1] [] <- [2, 3] [6] <- [4] [5]

Example

So to mutate node 4 in the example above you could do this: 


 tree.getZipper().next().next().next().setElement(7).close()

Which would generate the tree: 


 1
 .......
 2 3   6
   ...
   7 5

Navigation

The API to the rose tree zipper is somewhat similar to ImListZipper but with some differences:

These methods:

  • next()

  • prev()

  • hasNext()

  • hasPrev()

work on the tree as a whole (rather than just one list of sub-trees). next() on a tree that has sub-trees will move to the first sub-tree; otherwise, if it has a next sibling node, it will move to that; otherwise it will recursively try to move next on the parent. This means that repeated calls to next( ) will iterate over the whole tree.

If you use prev() on a zipper at the first tree in its parent's list then it will return the parent rather than a list in the initial state.

next() and prev() will always return a zipper that has a focus.

These methods:

  • nextSibling()

  • prevSibling()

  • hasNextSibling()

  • hasPrevSibling()

work on the list that contains the focus (rather than the whole tree) and operate exactly like a list zipper.

To move to the parent tree you can use up() and to move down to the sub-trees you can use down() - which moves to before the first sub-tree.

prevSibling() will sometimes return a zipper that has no focus.

See the package summary for more details about zippers.

    • up

      public ImRoseTreeZipper<T> up()

      The "parent" zipper - incorporating any changes made to the current list.

      up() calls close() on the current list so that the parent zipper focus will be a tree that incorporates any mutations that you made to the sub-trees.

      This: 

      
 ImRoseTree<Integer> tree = ImRoseTree.withNodes(1, leaf(2), ImRoseTree.withElements(3, 4, 5), leaf(6));

      will create a tree like this: 

      
 1
 .......
 2 3   6
   ...
   4 5

      If we get a zipper on node 3 : 

      
 ImRoseTreeZipper<Integer> z = tree.getZipper();
 ImRoseTreeZipper<Integer> z3 = z.next().next();

      and then get a zipper on node 4 : 

      
 ImRoseTreeZipper<Integer> z4 = z3.next();

      If we go up from z4 without making changes we merely get z3 

      
 z4.up() == z3  =>  "true"

      but if we make changes ... we don't 

      
 z4.setElement(7).up() == z3  => "false"

      @throws NoSuchElementException if the zipper is focussed on the root node

      public ImRoseTreeZipper<T> next()

      The zipper whose focus is the next node in the tree.

      If the node has sub-trees then the next node is the first sub-tree.

      If the focus is a leaf node, then the next node is the next sibling node, if it exists. If no such sibling node exists we ask the zipper obtained from from up() for the next sibling node and continue recursively until one is found or we reach the root.

      public ImRoseTreeZipper<T> prev()

      The zipper whose focus is the previous node in the tree.

      If the node has no previous sibling then the zipper obtained from up() is returned if one exists.

      If the node has a previous sibling then, if it is a leaf node, then it is returned.

      Otherwise, we return the previous node's last sub-tree node, if it is a leaf node, or continue recursively searching its sub-trees until we find a leaf node.

      @throws NoSuchElementException if the focus of this is the root of the tree
