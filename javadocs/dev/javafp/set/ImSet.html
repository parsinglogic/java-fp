Class ImSet<T>

public class ImSet<T> extends Object implements HasTextBox, Iterable<T>, Serializable

An immutable version of java.util.Set .

Introduction

An ImSet is a collection of objects with the condition that, if an object o belongs to the set then, for any other member a of the set: 


 a.equals(o) == false

As with the Java Set implementation, we use the hashCode of each object.

We therefore assume that the objects in the set have "reasonable" implementations of hashCode() and equals()

A web search for the phrase always override hashCode when you override equals will provide more details of the issues involved.

ImSets cannot contain null .

Add/Remove/Find/Replace

Note that, since ImSet s are immutable, you can't literally add/remove/replace elements in a set. Rather, you create a new set which has the same elements as the previous set except that a certain element is added/removed/replaced.

You can "add/replace" elements to the set using add(T) add(T) replace(T) and "remove" them using remove(T).

You can test if an element is in the set using contains(T) or find(T).

Methods that find/add/replace elements are O (log(n)) where n is the size of the set and log is base 2.

Replacing Elements

Sometimes it is convenient to be able to add an element to a set even though that set already contains an element that is "equal to it".

For example, in this library ImMap is implemented as a set of ImMap.Entry objects where an entry has a key and a value. Two ImMap.Entry objects are equal iff their keys are equal. So for any pair of entries e1 and e2 : 


 e1.equals(e2) == e1.key.equals(e2.key)

When we want to replace an entry, we want the new entry with its new value to be inserted in the set, even though there is an existing entry with the same key.

To allow this, there are methods replace(T) and add(T). The default behaviour when using add(T) is to not replace

However, this is a rather subtle distinction and we anticipate that, for most uses of ImSet , it is not important.

Implementation

An ImSet is a sorted set of buckets where elements whose hash codes are the same are stored in the same bucket.

Buckets are sorted on the hash value of their elements.

To find an element, we first find the bucket with the matching hash value and then look through the bucket to determine if the element is present.

For sets of elements with reasonable hash functions, the average bucket size will be one.

    • foldl

      public <B> B foldl(B z, Fn2<B,T,B> f)

      Start with an accumulator z and iterate over this , applying f to the z and e to get a new z

      One way to visualise this is to imagine that the function that we are using is the function that adds two numbers - ie the infix + operator

      Then 

      
 foldl (+) z [e1, e2, ... en] == [ (...((z + e1) + e2) + ... ) + en ]

      Note that the accumulator, z is the first argument to the function.

      If we extend this to imagine that the function is called * and can be applied using infix notation - like + then 

      
 foldl (*) z [e1, e2, ... en] == [ (...((z * e1) * e2) * ... ) * en ]

      Note that we are not assuming that * is commutative
